# 一、原则及说明

# 原则

众圈小程序是赋予开发者一种新的开放能力，可以在众圈合作的应用程序内被便捷地获取和传播，同时具有出色的使用体验。为保护用户、开发者权益及众圈小程序平台（以下简称“小程序平台”）安全，并方便小程序开发者（以下简称“您”或“开发者”）对平台审核规则进行了解，我们制订本小程序运营审核规范（以下简称“本规范”）。

众圈小程序需符合众圈平台的调性，给用户提供良好的体验。众圈小程序应提供具有价值的内容，遵守国家相关法律法规，不提供违法或违反相关规则的内容或从事相关行为；不从事过度营销行为，鼓励广大开发者向用户提供符合需求的内容或实用工具。

# 相关条款说明

为更清晰地理解并遵守相关协议及规则，小程序开发者必须阅读并遵守《众圈小程序平台服务协议》等专项规则，本运营规范基于上述协议及规则进行解释和说明，而非修改或变更上述协议及规则中的任何条款，目的是便于运营者能更顺利地使用众圈小程序平台的服务。

# 二、运营规范

# (一) 基本信息规范

# 1. 小程序注册规范

# 1.1 企业开发者

1.1.1 企业开发者除提供委托人（平台管理员）身份证正反面照片外，还需要提供年检内的企业营业执照副本照片

1.1.2 企业名称、企业执照注册号等须填写正确

1.2.3 企业执照不存在过期或存在经营异常、违法失信等情况

# 2. 小程序规范

小程序名称、图标、服务简介、小程序截图等基本信息审核规范。

# 2.1 小程序名称简介

2.1.1 名称可以由中文、数字、英文及下划线组成，长度控制在4-20个字符之间

2.1.2 名称不能以大类命名，如以电话、邮件、日历、等广义归纳类、或者其他普遍且不具有识别性词语来命名。示例：名称不是是单词汇，必须是两个词以上的组合

2.1.3 名称必须符合法律法规，无色情低俗、侵权、暴力等非法字符；且不能出现政治倾向、政治色彩

2.1.4 名称中不能出现已有知识产权的品牌logo、常用图形、产品、业务词汇相同

2.1.5 名称不能使用他人（企业）已经注册过的、不能使用他人（企业）已有知识产权的

2.1.6 在小程序发布/更新时可修改名称，每年可以修改两次

2.1.7 小程序简介明确介绍小程序的功能点，不能使用模糊的词义表达；小程序发布/更新时可修改简介

2.1.8 名称、简介的信息表达的意思必须有关联，具有一致性，并应与实际提供的功能一致，不含有与功能无关的搜索热词。示例：简介中能找到小程序名称或者分拆出来的词汇

# 2.2 小程序图标

2.2.1 小程序图标应清晰可辨。图标大小144px*144px，图片格式必须为：png,bmp,jpeg,jpg,gif；如：文字、物体、形状等将予以不通过

2.2.2 严禁使用二维码作为小程序图标，如果使用人像作为小程序图标需确认获得肖像权人授权证明。

2.2.3 小程序图标不允许出现违法、色情低俗等非法字符和图案。如不能出现“涉黄赌毒、黑卡、烟草等”；不能出现政治倾向、政治色彩

2.2.4 小程序图标不能使用他人（企业）已经注册过的，不能使用他人（企业）已有知识产权的

2.2.5 小程序图标不允许包含官方品牌标识

2.2.6 小程序图标不允许出现【认证】【第一】【官方】【国家】等误导性标签

# 2.3 小程序版本号规范及建议管理策略

2.3.1 小程序版本号规范：主版本号 . 子版本号 [ .修正版本号 ]；英文对照：Major_Version_Number.Minor_Version_Number[.Revision_Number]；示例：1.4.1

2.3.2 项目初版本时，版本号可以为 0.1.0 或 1.0.0

2.3.3 当项目在进行了局部修改或 bug 修正时，主版本号和子版本号都不变，修正版本号加 1

2.3.4 当项目在原有的基础上增加了部分功能时，主版本号不变，子版本号加 1，修正版本号复位为 0

2.3.5 当项目在进行了重大修改或局部修正累积较多，而导致项目整体发生全局变化时，主版本号加 1

# 2.4 小程序截图和服务类目

2.4.1 小程序需要提供3张截图，尽可能涵盖小功能的主体界面；截图中须至少1张展示软件运行中的画面（避免只截图启动画面或菜单）；截图不得有违反国家法律法规及尺度过大图片

2.4.2 小程序所设置的服务类目，应与所选的服务范围保持一致，类目不能超出服务范围。示例：服务范围是家政，服务类目是美食

# 2.5 功能设置规范

2.5.1 小程序所实际提供的服务和内容，需与小程序的简介一致，且不存在隐藏类目。小程序的服务范围需与实际填写的类目和标签一致，不应超出小程序平台的类目库范围。

2.5.2 小程序的核心功能，必须在小程序首页得到体现。

2.5.3 小程序向用户提供基本的功能指引，应包括描述或截图等形式。

2.5.4 未经众圈授权的情况下，不得提供与众圈客户端功能相同或者相似的功能。

2.5.5 未经允许或未经众圈授权的情况下，小程序功能的使用，不得依赖于其他小程序，即不得以使用其他小程序为条件，也不得依赖于其他应用程序，即不得以使用其他应用程序为条件；未经允许或未经众圈授权的情况下，也不得展示或者推荐其他小程序等。

2.5.6 小程序应设置侵权投诉机制，处理用户间的权益纠纷。

# 2.6 行为规范

2.6.1 小程序不得存在恶意刷量、刷单等行为。

2.6.2 不得在用户不知情或用户未进行任何触发操作的情况下跳转其他小程序。

2.6.3 不得使用或推荐插件、外挂或其他违规第三方工具、服务接入本服务和相关系统。

2.6.4 小程序不得滥用模板消息，包括但不限于利用模板消息骚扰用户和广告营销。

2.6.5 完成注册后，如帐号长期未登录，小程序可能被终止使用，终止使用后注册所使用的邮箱、身份证等信息可能将被取消注册状态。

2.6.6 不得通过小程序实施多级分销欺诈行为，发布分销信息诱导用户进行添加、分享或直接参与。

2.6.7 开发者应当正当使用众圈提供的服务，不得自行或与其他第三方共同对众圈提供的服务进行干扰或不正当使用，包括但不限于替换跳转页面、流量劫持等。

# 2.7 商标与商业外观

2.7.1 小程序必须遵守商标权、著作权等知识产权法律法规以及众圈关于知识产权使用的相关规则。

2.7.2 使用他人商标、版权内容等涉及他人知识产权的内容需要在帐号申请时如实说明，并根据要求提供相关权利证书或授权证明等。未经权利人授权，不得使用他人合法享有的知识产权的内容。

2.7.3 非众圈官方帐号，禁止在帐号名称、输出内容中出现与众圈已有知识产权内容相同，如“众圈”等相近似的中英文字样，或者容易与目前已有众圈产品设计主题、外观等相混淆的内容。

2.7.4 众圈在小程序注册入驻中为您提供的内容（包括但不限于网页、文字、图片、音频、视频、图表等）的知识产权归两码事公司所有，用户在使用小程序中所产生的内容的知识产权归用户或相关权利人所有。

# (二) 小程序质量规范

# 1. 小程序可用性和稳定性规范

# 1.1 可用性

1.1.1 提交的小功能须是一个完成品，要求可以打开，可以运行

1.1.2 按钮使用符合UI涉及规范（类型、颜色、尺寸）

1.1.3 小程序二维码须可以扫码测试且测试的小程序和提交的名称等保持一致

# 1.2 稳定性

1.2.1 提供的小程序本身会崩溃，或小程序不能造成众圈相关其他客户端崩溃

1.2.2 小程序不能存在严重Bug的小程序（如无法添加二维码和打开、无法返回和退出、卡顿严重等）

1.2.3 若小程序中存在帐号体系，需提供测试号，包含帐号和密码（可以体验所有功能）

1.2.4 小程序出现网络异常、加载超时、系统错误等情况时需给予用户相应提示，不可出现闪退等其他错误

1.2.5 小程序接口问题，对接众圈相关开放平台产品需遵循该产品接口接入规范，使用小程序API接口功能，需确保各个API接口调用功能正确

1.2.6 小程序有需要用户输入填写的操作，填写、提交功能须正常，可以正常收到验证码等

1.2.7 电商等小程序，订单的添加、确认、支付、修改、退订等须正常可以使用

1.2.8 支付环节可成功支付，支付完成后跳转正常

1.2.9 登陆除默认主端账号外，第三方账号须能正常退出登陆的逻辑，且保证可跳转

# 2. 小程序性能测试规范

# 2.1 小程序运行情况

2.1.1 小程序本身不存在崩溃、闪退等情况

2.1.2 小程序不会引起众圈相关合作APP 崩溃、闪退

# 2.2 小程序响应时间

2.2.1 小程序页面加载、页面按钮响应时间不能过长；流量、WIFI 场景下使用无异常

2.2.2 小程序操作流畅、无卡顿

2.2.3 有服务端情况下，需要关注小程序并发性（QPS*、*TPS等）问题。承载流量和服务器的承受能力

# 2.3 小程序兼容性

2.3.1 兼容的机型。如小米、vivo、Huawei mate 、Huawei P10 ，iPhone 系列

2.3.2 系统版本。如Android 4.x（可选）/Android 5.x/Android 6.x/Android 7.0等；iOS 9.x/iOS 10.x等

2.3.3 众圈相关合作app版本。业务方需要兼容众圈相关合作APP最新版本以及低版本等

# 3. 其他规范

# 3.1 涉及付费权限

3.1.1 付费的服务拨打电话，直接拨打电话号码

3.1.2 付费的服务发送短信（重要权限，恶意程序可秘密发送短信，产生费用）

3.1.3 更改网络连接性（在系统工具中）

# 3.2 涉及隐私权限

3.2.1 付费的服务写入、编辑或者删除短信或接收短信（重要权限，可能导致信息被监控或被删除）

3.2.2 您的信息读取短信或彩信（重要权限，恶意程序可获取机密信息）

3.2.3 手机通话读取用户数据手机状态和身份（重要权限，恶意程序可获取手机号码和IMEI等）

3.2.4 您的个人信息读取联系人数据 隐私权限

3.2.5 您的位置（基于网络的）粗略位置隐私权限

3.2.6 您的位置精准的(GPS)位置隐私权限

注意：辨别的时候除了那些非常明显予以排除，关键要根据软件的实际用途再判断拥有该权限是否正常。

图书单行本、壁纸、主题、锁屏类软件 不能有敏感权限：“拨打电话”，“发送短信”，"读取联系人数据"隐私权限。若小功能描述内有注明敏感权限用途且符合软件本身需求，可允许上架。

# 3.3 技术性规范

3.3.1 需要提供小程序文档和说明

3.3.2 禁止视频、音乐、语音等多媒体的自动播放

3.3.3 安装或运行其他可执行代码的程序，将会被拒绝

3.3.4 违规加载或更新代码，将会被拒绝

3.3.5 如果小程序有账户系统，必须提供能正常使用且易于发现的“退出”账户选项

# 3.4 UI规范

3.4.1 小程序页面内的浮层和弹窗可关闭

3.4.2 小程序的界面必须遵守众圈相关平台的外观和功能，不得提供改变众圈相关平台外观和功能的产品体验

3.4.3 小程序的界面不得模仿系统通知或警告诱导用户点击

3.4.4 小程序图标ICON需使用透明或有色背景。若使用白色背景，需使用有色边框

3.4.5 如相关法律法规规定提供服务的界面必须进行特别信息标识的，应予以明确标识。

# 3.5 用户隐私和数据规范

3.5.1 小程序的数据收集，存储与授权，须遵循包含但不仅限于以下规范：

在采集用户数据之前，必须确保经过用户明示同意授权，并向用户如实披露采集数据内容，数据用途、使用范围等相关信息。

不得非法收集或窃取用户密码或其他个人数据。

除非相关法律要求，或经用户同意，否则小程序不得强迫用户输入个人信息（包括但不限于，姓名，手机号、身份证号、生日、住址等）才可以使用其功能，或收集用户密码或者用户个人信息（包括但不限于，手机号、身份证号、生日、住址等）。

若用户要求，小程序服务提供者应该删除接收的所有关于该用户的数据，除非依据法律、法规，小程序服务提供者有义务保留这些数据。

禁止为任何用户自动登录到众圈合作的任意平台提供代理身份验证凭据。

禁止使用任意方式非众圈授权许可的方式爬取众圈相关合作应用程序内的信息。

用户拒绝授权后，小程序服务提供者有义务清除并不再继续使用该用户的头像、昵称、手机号、电子邮箱地址等数据。

禁止小程序首屏引导用户授权，需在用户充分了解小程序业务的前提下，自主决定是否允许授权；

禁止小程序多页面重复授权相同信息（包括但不限于昵称、头像、生日等）

3.5.2 小程序的数据使用规范：

不允许向小程序其他用户或任何第三方显示用户个人隐私包括但不限于众圈相关合作应用程序账号、名称、手机号、电子邮箱地址等信息。

不允许将用户信息，包括但不限于众圈旗下应用程序账号、名称、手机号、电子邮箱地址和出生日期等信息用于任何未经用户及众圈平台授权的用途。

不允许进行反射查找、跟踪、关联、挖掘、获取或利用用户个人信息包括但不限于众圈旗下应用程序账号、名称、手机号、电子邮箱地址等信息从事与小程序所公示身份无关的行为。

不允许在以下场景复制、存储、使用或传输用户数据，包括但不限于未经用户明确同意、未向用户如实披露数据用途、使用范围等相关信息。

通过众圈接收的数据，需保持数据为最新数据，并利用这些数据改善小程序的用户体验。

未经众圈授权或允许，不得使用从众圈和小程序接收的数据用于做出有关资格的决定，包括决定是否批准或拒绝某项申请，或对某项贷款收取多少利息等。

若小程序服务提供者的主体被第三方收购或合并，则小程序服务提供者从众圈和小程序接收的数据仅能在小程序内继续使用。

在检测到相关小程序遭到非法攻击产生的异常状况或数据泄露后，众圈有权对相关接口进行保护，包括但不限于临时关闭，限制流量，限制频次等方式阻断恶意攻击。

禁止明文传输用户隐私数据。

3.5.3 小程序数据安全：

请谨慎保管好所使用的帐号、密码和密钥。

若使用第三方合作伙伴服务，那么应同合作伙伴签署合同，以保护从众圈获取的任何信息，限制其对这些信息的使用并保持信息的保密性。

不得要求用户降低手机操作系统安全性（如要求iPhone 用户越狱、Android 用户ROOT 等）后，方能使用相关功能。小程序有义务保护用户授权的隐私信息与数据。

一旦小程序服务提供者的停止使用众圈提供的相关服务，或众圈基于任何原因终止小程序服务提供者使用相关服务，必须立即删除全部因使用本服务而获得的数据（包括各种备份）， 且不得再以任何方式继续使用。

当出现任何产生数据泄露的情况，例如：相关系统被渗透入侵，小程序服务提供者有义务通知众圈。众圈有义务配合进行紧急安全处理与相关调查。

在特定小程序中收集的用户数据仅可以在该特定小程序中使用，不得将其使用在该特定小程序之外或为其他任何目的进行使用，即使你可能同时注册了两个以上的小程序。

小程序所有展示的用户个人隐私信息（包括但不限于姓名，手机号、身份证号），必须进行脱敏。

3.5.4 小程序地理位置：

在采集、传送或使用地理位置数据之前必须获得用户同意，且地理位置数据只能用于该小程序提供的直接相关功能或服务，不得公开展示给其他用户。

地理位置数据只能用于小程序提供的直接相关功能或服务。

# 3.6 小程序支付规范

3.6.1 若小程序内存在支付行为，需在小程序的简介中有明确的指示。

3.6.2 执行支付动作前，需在页面中对用户有明确的提示。

3.6.3 在小程序平台使用支付功能的，应当依照法律、法规等规范性文件的规定开展，不得实施非法吸收公众存款、集资诈骗等违法犯罪行为，或为违法犯罪行为提供协助

# (三) 小程序内容规范

# 1. 小程序的页面内容规范

# 1.1 不能存在诱导、严广、封建迷信等行为

1.1.1 包括但不限于诱导分享、诱导添加、诱导关注第三方平台、诱导下载等，要求用户分享、添加、关注或下载后才可操作的程序，含有明示或暗示用户分享的文案、图片、按钮、浮层、弹窗等的小程序 ，通过利益诱惑诱导用户分享、传播的小功能 ，用夸张言语来胁迫、引诱用户分享 ，强制或诱导用户添加小程序，都将会被拒绝

1.1.2 小程序的页面内容中，主要为营销或广告用途（如内含空白广告位、招商广告等），将会被拒绝；示例：漂浮悬浮广告，含有功能使用的页面中的广告展示比例超过50%，广告遮挡功能

1.1.3 小程序的页面内容中，存在对用户产生误导、引发用户恐惧心理、严重破坏用户体验或损害用户利益的谣言类等内容的，将会被拒绝

1.1.4 小程序的页面内容中，不能存在封建迷信类内容；示例：算命等

1.1.5 小程序的页面内容中不能存在虚假、欺诈类内容，包括但不限于虚假红包、虚假活动、宣传或销售侵害他人合法权益的商品；仿冒众圈官方或他人业务，其他可能造成用户混淆的内容和服务等；页面中不能存在误导和错误暗示众圈公司与该小功能有任何合作、投资、背书关系的内容

1.1.6 小程序的页面中不能含有传播骚扰信息、广告信息和垃圾信息等内容

1.1.7 小程序内的图片上不能含有广告、网址或虚假内容

1.1.8 小程序的页面中不得含有可能违反与众圈签订的、任何形式的服务协议、平台协议、功能协议的内容

# 1.2 不能含有发布、传送、传播、储存违反国家法律法规的或含有以下信息内容的行为

1.2.1 反对宪法所确定基本原则的，危害国家安全、泄露国家秘密、颠覆国家政权、破坏国家统一 、损害国家荣誉和利益的小程序；

1.2.2 任何带有虚假、欺诈内容等的小程序不予通过；

1.2.3 任何召集、推销、鼓动犯罪或有明显侵犯社会善良风俗行为的小程序不予通过；

1.2.4 任何包含法律法规禁止传播内容的小程序不予通过；

1.2.5 小程序内容包含反政府、反社会或不符合主流政治的行为的，或存在煽动性的涉政言论或国家法律禁止的内容的，或含有散布谣言，扰乱社会秩序，破坏社会稳定信息的，不予通过；

1.2.6 小程序内容不能含有色情素材（即旨在激发情欲，对性器官或性行为的明确描述或展示，而无关美学），或存在涉嫌宣扬传播淫秽、色情内容信息，包括暴露图片、挑逗内容等的，或包含非法色情交易的信息；

1.2.7小程序内容不能包含煽动民族仇恨、民族歧视、破坏民族团结的内容、破坏国家宗教政策、宣扬邪教和封建迷信的；

1.2.8小程序内容不能包含展示人或动物被杀戮、致残、枪击、针刺或其他伤害的真实图片，描述暴力或虐待儿童的，或包含宣扬暴力血腥内容的，或包含侮辱或者诽谤他人，侵害他人合法权益信息的，将会被拒绝；

1.2.9 小程序内容不能包含赌博、有奖竞猜和抽奖的内容，提供游戏类目服务的小功能如采取随机抽取方式提供虚拟道具和增值服务的，不得诱导用户以直接投入法定货币或者虚拟货币的方式参与；

1.2.10 小程序内容不能包含诱发未成年人违反公德、违法犯罪的行为的内容以及妨害未成年人身心健康的内容。

# 三、处罚规范

# 目的与原则

小程序是赋予开发者一种新的开放能力，可以在多个城市公交APP内被便捷地获取和传播，同时具有出色的使用体验。为保护用户、开发者权益及小程序平台安全，两码事公司制定《小程序违规处理规则》(以下简称“本规则”)。本规则作为小程序开发者与两码事公司签署的协议（以下简称“平台服务协议”）的一部分，与平台主服务协议具有相同的法律效力。

# 处罚措施

小程序开发者指通过其开发的移动化工具即小程序，为用户提供应用或服务的业务主体，可以是个人或者企业。

审核拒绝： 指平台方通过审核规则规范，对于存在不符合平台运营审核规范的小程序拒绝，不予通过。

沟通警告： 指平台方通过邮件、电话等形式对开发者的不当行为进行警告，要求开发者在指定的时限内完成自查及整改，以使经营业务符合国家法律法规、政策、规章的规定、监管要求、平台服务协议约定以及本规则规定。

限期整改： 指平台方依据本规则对违规小程序进行单个版本下架整改，单个版本下架后用户不可见，用户不可使用，但开发者可继续创建新版本直到符合要求后可再提交。

账号封禁： 如果小程序存在不同程度的违规违法情况，会依据平台规则取消相应的部分平台权限，包括且不限于下架小程序，情节恶劣严重者封号处理。
