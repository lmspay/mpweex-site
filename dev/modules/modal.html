# modal

modal 模块提供了以下展示消息框的 API：toast、alert、confirm、 prompt 和progress。

# API

# toast

toast() 会在一个小浮层里展示关于某个操作的简单反馈。例如，在邮件发送前离开邮件编辑界面，可以触发一个“草稿已保存”的 toast，告知用户以后可以继续编辑。toast 会在显示一段时间之后自动消失。

# toast(options)

  • @options
    • message, string, 展示的内容.
    • gravity, string, 展示的位置，支持top、center、bottom三个值
    • duration, number, 持续时间（以秒为单位）

TIP

  • Android: 如果时间长度大于3s，将使用一个被称为LONG的系统变量，否则使用SHORT这个系统变量
  • iOS: 持续的时间与duration相同
var modal = weex.requireModule('modal')
modal.toast({
    message: 'This is a toast',
    duration: 0.3
})

# alert

警告框经常用于确保用户可以得到某些信息。当警告框出现后，用户需要点击确定按钮才能继续进行操作。

# alert(options, callback)

  • @options
    • title, string，警告框内显示的标题
    • message, string，警告框内显示的文字信息
    • okTitle, string，确认按钮上显示的文字信息
  • @callback, function，用户操作完成后的回调
var modal = weex.requireModule('modal')
modal.alert({
  message: 'This is a alert',
  okTitle: '确认'
}, function () {
  console.log('alert callback')
})

# confirm

确认框用于使用户可以验证或者接受某些信息。当确认框出现后，用户需要点击确定或者取消按钮才能继续进行操作。

# confirm(options, callback)

  • @options
    • title, string，警告框内显示的标题
    • message, string，警告框内显示的文字信息
    • okTitle, string，确认按钮上显示的文字信息
    • cancelTitle, string，取消按钮上显示的文字信息
  • @callback, function，用户操作完成后的回调，参数是按下按钮上的文字信息
    • result, string, 用户按下的按钮, 固定返回cancel、success两个值
var modal = weex.requireModule('modal')
modal.confirm({
    message: 'Do you confirm ?',
    okTitle: 'ok label',
    cancelTitle: 'cancel label'
}, function (value) {
    console.log('confirm callback', value)
})

# prompt

提示框经常用于提示用户在进入页面前输入某个值。当提示框出现后，用户需要输入某个值，然后点击确认或取消按钮才能继续操作。

# prompt(options, callback)

  • @options
    • title, string，警告框内显示的标题
    • message, string，警告框内显示的文字信息
    • hint, string，输入框内的placeholder
    • default, string，输入框内的默认值，可选
    • okTitle, string，确认按钮上显示的文字信息
    • cancelTitle, string，取消按钮上显示的文字信息
  • @callback, function，用户操作完成后的回调
    • res.result：用户按下的按钮, 固定返回cancel、success两个值
    • res.data：用户输入的文字信息
var modal = weex.requireModule('modal')
modal.prompt({
    message: 'This is a prompt',
    hint: 'hint text',
    okTitle: 'ok label',
    cancelTitle: 'cancel label'
}, function (value) {
    console.log('prompt callback', value)
})

# progress

用于显示加载提示框，分可取消及不可取消两种，通过参数控制。可以通过modal.dismiss()取消。

# progress(options, callback)

  • @options
    • message, string，加载时的文字提示
    • cancelable, boolean，用户是否可取消，默认为false
  • @callback, function，如果用户取消了，则会回调，正常dismiss不会回调。
var modal = weex.requireModule('modal')
modal.progress({
    // cancelable: false,
    message: '正在加载...'
}, function (value) {
    modal.toast({
        message: `${value}`
    });
});

setTimeout(function () {
    modal.dismiss();
}, 2000);
